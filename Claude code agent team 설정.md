Claude Code Agent Teams 설정 및 사용법

  1단계: 활성화 (완료)

  ~/.claude/settings.json에 아래와 같이 설정:

  {
    "env": {
      "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
    }
  }

  2단계: 디스플레이 모드 선택 (선택사항)
  ┌─────────────┬──────────────┬─────────────────────────────────────────────────┐
  │    모드     │      값      │                      설명                       │
  ├─────────────┼──────────────┼─────────────────────────────────────────────────┤
  │ In-process  │ "in-process" │ 메인 터미널 안에서 실행. Shift+Up/Down으로 전환 │
  ├─────────────┼──────────────┼─────────────────────────────────────────────────┤
  │ Split panes │ "tmux"       │ tmux/iTerm2에서 별도 패널로 실행                │
  ├─────────────┼──────────────┼─────────────────────────────────────────────────┤
  │ Auto (기본) │ "auto"       │ tmux 세션이면 분할, 아니면 in-process           │
  └─────────────┴──────────────┴─────────────────────────────────────────────────┘
  Windows 사용자: Split panes는 Windows Terminal에서 지원되지 않으므로 in-process 모드를 사용하세요.

  설정 예시:
  {
    "env": {
      "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
    },
    "teammateMode": "in-process"
  }

  3단계: 사용 방법

  Claude Code를 재시작하면 TeammateTool이 활성화됩니다. 자연어로 지시하면 됩니다:

  > 이 프로젝트의 버그를 3명의 에이전트로 나눠서 수정해줘
  > 에이전트 팀을 만들어서 리서치, 코딩, 테스트를 병렬로 진행해줘

  주요 조작 키
  ┌───────────────┬────────────────────────────────────┐
  │      키       │                동작                │
  ├───────────────┼────────────────────────────────────┤
  │ Shift+Up/Down │ 팀원 간 전환                       │
  ├───────────────┼────────────────────────────────────┤
  │ Enter         │ 선택한 팀원 세션 보기              │
  ├───────────────┼────────────────────────────────────┤
  │ Escape        │ 팀원 인터럽트                      │
  ├───────────────┼────────────────────────────────────┤
  │ Ctrl+T        │ 작업 목록 토글                     │
  ├───────────────┼────────────────────────────────────┤
  │ Shift+Tab     │ 위임 모드 (리더가 직접 코딩 안 함) │
  └───────────────┴────────────────────────────────────┘
  작동 구조

  Team Lead (메인 세션)
    ├── Teammate A (Explore) ── 코드 분석
    ├── Teammate B (general-purpose) ── 구현
    └── Teammate C (Bash) ── 테스트 실행

  - Team Lead: 작업 분배, 결과 종합
  - Teammates: 독립된 Claude Code 인스턴스, 각자 별도 컨텍스트
  - 공유 작업 목록: 의존성 관리 포함 (A 완료 후 B 시작 등)
  - 메시지 시스템: 에이전트 간 직접 소통 가능

  알아둘 제한사항

  - 세션 재개(/resume) 시 팀원은 복원되지 않음
  - 팀 중첩 불가 (팀원이 또 다른 팀을 만들 수 없음)
  - 리더 변경 불가
  - Windows에서는 split pane 미지원 → in-process 모드 사용
